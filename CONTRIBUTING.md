# Contributing to MARS Studio

First off, thank you for considering contributing to MARS Studio! It's people like you that make MARS Studio a powerful educational tool for Computer Architecture.

## Getting Started

1. **Fork the repository** on GitHub.
2. **Clone** your fork locally: `git clone https://github.com/tahanawab4848/mars-mips-simulator.git` (or your fork's URL).
3. **Open the project** in your preferred IDE (like IntelliJ IDEA or Eclipse).

## Building the Project

MARS Studio uses Gradle for its build system. To build and run the project locally:

**Windows:**
```bash
.\gradlew.bat run
```

**Linux/macOS:**
```bash
./gradlew run
```

## How to Contribute

### Reporting Bugs
If you find a bug, please create an Issue on GitHub. Include:
- Your operating system and Java version.
- Steps to reproduce the bug.
- Any relevant logs or error messages.

### Suggesting Enhancements
Have an idea for a new visualizer or compiler feature? Open an Issue tagged as an "Enhancement" and describe your idea in detail.

### Submitting Pull Requests
1. Create a new branch for your feature or bugfix: `git checkout -b feature-name`
2. Commit your changes with descriptive commit messages.
3. Push to your fork and submit a Pull Request against the `master` branch of the main repository.
4. Ensure your code follows the existing style and doesn't break core emulation features.

Thank you for helping modernize Computer Architecture education!
